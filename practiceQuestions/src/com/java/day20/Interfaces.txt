In Java, interfaces are a way to achieve abstraction and multiple inheritance. They define a contract that a class must follow.

🔹 1. What is an Interface?

An interface is a reference type in Java that contains:

Abstract methods (by default)

Constants (public static final variables)

Default methods (Java 8+)

Static methods (Java 8+)

Private methods (Java 9+)

Syntax:
interface Animal {
    void sound();   // abstract method
}
🔹 2. Implementing an Interface

A class uses the implements keyword to implement an interface.

interface Animal {
    void sound();
}

class Dog implements Animal {
    public void sound() {
        System.out.println("Bark");
    }
}

⚠ Important:

The method must be public

The class must implement all abstract methods

🔹 3. Why Use Interfaces?
✅ Achieve Abstraction

Only method declaration, no implementation (mostly).

✅ Multiple Inheritance

Java does not support multiple inheritance with classes, but it does with interfaces.

interface A {
    void methodA();
}

interface B {
    void methodB();
}

class C implements A, B {
    public void methodA() {}
    public void methodB() {}
}
🔹 4. Interface vs Class
Feature	Interface	Class
Methods	Abstract by default	Can have concrete methods
Variables	public static final	Normal variables
Constructors	❌ No	✅ Yes
Multiple Inheritance	✅ Yes	❌ No
🔹 5. Default Methods (Java 8+)

Interfaces can have methods with implementation using default.

interface Vehicle {
    default void start() {
        System.out.println("Vehicle starting...");
    }
}
🔹 6. Static Methods in Interface
interface MathUtils {
    static int add(int a, int b) {
        return a + b;
    }
}

Call using:

MathUtils.add(5, 3);
🔹 7. Real-World Example
interface Payment {
    void pay(double amount);
}

class CreditCardPayment implements Payment {
    public void pay(double amount) {
        System.out.println("Paid " + amount + " using Credit Card");
    }
}
🔹 8. Marker Interface

An interface with no methods.

Example:

interface Serializable {
}

Used to "mark" a class.

🔹 Key Points to Remember

Interface methods are public and abstract by default.

Variables are public static final by default.

A class can implement multiple interfaces.

Interfaces support abstraction and loose coupling.

A real-time example of an Interface in Java is commonly seen in Payment Systems, Notification Systems, or Device Drivers. Let’s take a clear real-world payment gateway example that companies like e-commerce platforms use.

Real-Time Example: Payment System 💳

Imagine an online shopping application where users can pay using:

Credit Card

UPI

PayPal

The payment process is common, but the implementation is different.
So we define a Payment Interface.

1️⃣ Interface (Common Contract)
interface Payment {
    void pay(double amount);
}

This means every payment method must implement pay().

2️⃣ Credit Card Implementation
class CreditCardPayment implements Payment {

    public void pay(double amount) {
        System.out.println("Paid " + amount + " using Credit Card");
    }
}
3️⃣ UPI Implementation
class UpiPayment implements Payment {

    public void pay(double amount) {
        System.out.println("Paid " + amount + " using UPI");
    }
}
4️⃣ PayPal Implementation
class PaypalPayment implements Payment {

    public void pay(double amount) {
        System.out.println("Paid " + amount + " using PayPal");
    }
}
5️⃣ Main Class
public class OnlineStore {

    public static void main(String[] args) {

        Payment payment;

        payment = new CreditCardPayment();
        payment.pay(5000);

        payment = new UpiPayment();
        payment.pay(2000);

        payment = new PaypalPayment();
        payment.pay(3500);
    }
}
Output
Paid 5000 using Credit Card
Paid 2000 using UPI
Paid 3500 using PayPal
Real-World Concept Behind This

In real applications (Amazon, Flipkart, etc.):

Payment (Interface)
       |
--------------------------------
|              |               |
CreditCard     UPI           PayPal

The system only knows "Payment", not the specific payment type.

So new methods like:

Apple Pay

Google Pay

Net Banking

can be added without modifying existing code.

Why Interface is Used Here

✔ Flexibility
✔ Loose Coupling
✔ Easy to add new payment methods
✔ Code maintainability

We will build a Payment Gateway System (very common real-world scenario).

Real-Time Example: Payment Gateway Interface
1️⃣ Interface
interface PaymentGateway {

    // public static final variable (constant)
    double TRANSACTION_FEE = 2.5;

    // abstract method
    void makePayment(double amount);

    // default method
    default void printReceipt(double amount) {
        validateTransaction(amount);
        System.out.println("Receipt Generated for amount: " + amount);
    }

    // static method
    static void gatewayInfo() {
        System.out.println("Welcome to Universal Payment Gateway");
    }

    // private method (used internally)
    private void validateTransaction(double amount) {
        if (amount <= 0) {
            System.out.println("Invalid Transaction Amount");
        } else {
            System.out.println("Transaction Validated");
        }
    }
}
2️⃣ Implementation Class
class CreditCardPayment implements PaymentGateway {

    public void makePayment(double amount) {
        double total = amount + TRANSACTION_FEE;
        System.out.println("Processing Credit Card Payment...");
        System.out.println("Total amount including fee: " + total);
    }
}
3️⃣ Another Implementation
class UpiPayment implements PaymentGateway {

    public void makePayment(double amount) {
        System.out.println("Processing UPI Payment of " + amount);
    }
}
4️⃣ Main Class
public class PaymentSystem {

    public static void main(String[] args) {

        PaymentGateway.gatewayInfo(); // static method call

        PaymentGateway payment1 = new CreditCardPayment();
        payment1.makePayment(1000);
        payment1.printReceipt(1000);

        System.out.println("------------------");

        PaymentGateway payment2 = new UpiPayment();
        payment2.makePayment(500);
        payment2.printReceipt(500);
    }
}
Output
Welcome to Universal Payment Gateway

Processing Credit Card Payment...
Total amount including fee: 1002.5
Transaction Validated
Receipt Generated for amount: 1000

------------------

Processing UPI Payment of 500
Transaction Validated
Receipt Generated for amount: 500
