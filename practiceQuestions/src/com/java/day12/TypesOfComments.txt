Types of comments and what is the usage in java

In Java, comments are used to explain code, improve readability, and generate documentation. They are ignored by the compiler.

There are three main types of comments in Java:

1️⃣ Single-Line Comment

Syntax:

// This is a single-line comment

Usage:

Used for short explanations

Explaining one line of code

Temporary debugging (commenting out code)

Example:

int x = 10; // Initialize x with value 10
2️⃣ Multi-Line Comment (Block Comment)

Syntax:

/* This is a
   multi-line comment */

Usage:

Writing longer explanations

Commenting multiple lines of code

Temporarily disabling a block of code

Example:

/*
  This program calculates
  the sum of two numbers
*/
int sum = a + b;
3️⃣ Documentation Comment (Javadoc Comment)

Syntax:

/**
 * This is a documentation comment
 */

Usage:

Used to generate API documentation

Written before classes, methods, or fields

Processed by the Javadoc tool

Example:

/**
 * Adds two integers and returns the result.
 * @param a first number
 * @param b second number
 * @return sum of a and b
 */
public int add(int a, int b) {
    return a + b;
}
Common Javadoc Tags:

@param → Describes method parameters

@return → Describes return value

@author → Author name

@version → Version info

@throws → Exception description
