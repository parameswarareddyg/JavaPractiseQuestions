3. Why String is Immutable

Java designers made String immutable for 4 main reasons.

1️⃣ Security

Strings are used in sensitive operations like:

Database URLs
File paths
Network connections
Usernames and passwords

Example:

String url = "jdbc:mysql://localhost:3306/db";

If strings were mutable:

Attacker could modify the URL

Example problem:

jdbc:mysql://localhost:3306/db
↓
jdbc:mysql://hacker-server

Immutable strings prevent this security risk.

2️⃣ String Pool Optimization (Memory Saving)

Java stores string literals in String Constant Pool.

Example:

String s1 = "Java";
String s2 = "Java";

Memory:

String Pool
--------------
"Java"  ← s1
        ← s2

Both references point to the same object.

If String were mutable:

Changing s1 would also change s2

Example:

s1 = "Java Programming"

Then:

s2 would also change

This would cause serious data corruption.

So immutability makes string pooling safe.

3️⃣ Thread Safety

Immutable objects are automatically thread-safe.

Example:

Two threads using same string:

String name = "Java";

Thread1:

reads "Java"

Thread2:

reads "Java"

Since it cannot change, there is no concurrency issue.

If mutable:

Thread1 modifies
Thread2 reads corrupted value

So immutability prevents race conditions.

4️⃣ HashCode Caching (Performance)

Strings are heavily used in HashMap, HashSet.

Example:

HashMap<String,Integer> map = new HashMap<>();
map.put("Java",1);

HashMap uses:

hashCode()

If string was mutable:

hashCode would change

Example:

"Java" → hashcode = 1234

If modified to:

"Java Programming"

New hashcode:

5678

Now HashMap cannot find the key.

So immutability ensures hashcode remains constant.

4. Memory Diagram

Example:

String s = "Java";
s.concat(" Programming");

Memory representation:

String Pool
----------------------

"Java"                ← s

"Java Programming"    (new object)

Original object never changes.

5. Example Showing Immutability
public class Test {

    public static void main(String[] args) {

        String s = "Java";

        s.concat(" Programming");

        System.out.println(s);

    }
}

Output

Java

Because:

concat() creates new object
but reference not stored

Correct usage:

s = s.concat(" Programming");

Output:

Java Programming
6. Interview-Level Answer (Short)

If interviewer asks:

Why String is immutable in Java?

You can answer:

String is immutable in Java for security, string pool optimization, thread safety, and performance improvements like hashcode caching.
7. Another Interview Trap Question

Interviewer may ask:

If String is immutable, how does this work?

String s = "Java";
s = s.concat(" Programming");

Answer:

A new String object is created and the reference variable is updated.
The original string remains unchanged.
